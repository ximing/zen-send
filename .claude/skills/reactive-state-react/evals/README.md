# Reactive State React Skill 评估测试

这个目录包含了 `reactive-state-react` skill 的评估测试用例。

## 测试覆盖范围

### 核心概念测试（5个核心主题）

1. **响应式组件测试** (Eval #1, #5)
   - 测试 `observer` 和 `view` 的使用
   - 测试常见错误（忘记 observer、解构破坏响应性）

2. **Service 测试** (Eval #1, #2, #10)
   - 测试 Service 类的创建
   - 测试异步方法和 `$model` 状态追踪
   - 测试计算属性（getter）

3. **useService + bindServices 测试** (Eval #1, #3)
   - 测试服务注册和使用
   - 测试容器生命周期绑定

4. **生命周期测试** (Eval #3)
   - 测试全局单例 vs 组件绑定的区别
   - 测试 `register` vs `bindServices` 的正确使用

5. **Service 依赖关系测试** (Eval #4, #7)
   - 测试 `resolve()` 的使用
   - 测试作用域链规则
   - 测试依赖注入模式

### 进阶功能测试

6. **装饰器测试** (Eval #6)
   - 测试 `@Debounce` 装饰器
   - 测试是否引导用户查看参考文档

7. **领域架构测试** (Eval #7)
   - 测试多级嵌套容器
   - 测试父子容器访问

8. **Hooks API 测试** (Eval #8)
   - 测试 `useObserverService` 细粒度追踪
   - 测试是否引导用户查看 hooks 文档

9. **事件系统测试** (Eval #9)
   - 测试全局事件通信
   - 测试容器事件 vs 全局事件的区别

10. **计算属性测试** (Eval #10)
    - 测试 getter 的使用
    - 测试自动依赖追踪

## 测试用例设计原则

### 1. 真实场景

所有测试用例都基于实际开发中的常见需求：

- 计数器（基础状态管理）
- 用户列表（异步数据加载）
- 主题服务（全局状态）
- 搜索功能（防抖）
- 购物车（计算属性）

### 2. 渐进式难度

- Eval #1-3: 基础用法
- Eval #4-7: 中级特性（依赖、作用域）
- Eval #8-10: 高级特性（细粒度追踪、事件、计算属性）

### 3. 错误诊断

- Eval #5 专门测试常见错误的识别和修复
- 测试 skill 是否能正确指导用户解决问题

### 4. 文档引导

- 测试 skill 是否在适当时机引导用户查看参考文档
- 确保基础用法在 SKILL.md，高级用法引导到 references/

## 断言设计

每个测试用例包含 3-5 个断言，覆盖：

1. **核心模式识别**
   - 是否使用了正确的 API（observer, useService, bindServices）
   - 是否遵循了最佳实践

2. **常见错误避免**
   - 是否避免了解构 observable
   - 是否正确处理异步状态
   - 是否正确注册依赖

3. **概念理解**
   - 是否理解生命周期差异
   - 是否理解作用域链规则
   - 是否理解事件系统范围

4. **文档引导**
   - 是否在适当时机引导用户查看参考文档

## 运行评估

使用 skill-creator skill 运行评估：

```bash
# 运行所有测试
claude skill-test reactive-state-react

# 运行特定测试
claude skill-test reactive-state-react --eval-id 1
```

## 预期结果

成功的 skill 应该：

1. ✅ **基础用法 100% 正确** (Eval #1-3)
   - 正确使用 5 大核心概念
   - 没有常见错误

2. ✅ **依赖管理清晰** (Eval #4, #7)
   - 正确使用 resolve()
   - 理解作用域链

3. ✅ **错误诊断准确** (Eval #5)
   - 能识别常见错误
   - 提供正确修复方案

4. ✅ **文档引导及时** (Eval #6-10)
   - 基础用法直接给出答案
   - 高级用法引导查看参考文档

## 改进建议

如果测试失败，可能需要：

1. **SKILL.md 内容调整**
   - 核心概念是否足够清晰
   - 示例代码是否足够完整
   - 常见错误是否有说明

2. **参考文档补充**
   - 是否缺少某些高级特性的文档
   - 文档链接是否正确

3. **断言优化**
   - 断言是否过于严格或宽松
   - 是否需要添加新的断言
